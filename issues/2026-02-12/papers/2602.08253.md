# G-LNS: 基于 LLM 的生成式大邻域搜索自动启发式设计

## 一句话总结
G-LNS 提出了一个生成式进化框架，利用 LLM 协同进化 LNS 的破坏-修复算子对，在 TSP 和 CVRP 上显著超越现有 LLM AHD 方法和经典求解器。

## 关键创新
- **LNS 算子自动设计**：将 LLM 的自动启发式设计扩展到大邻域搜索算子
- **破坏-修复算子协同进化**：LLM 同时进化紧密耦合的破坏和修复算子对
- **协作评估机制**：显式捕获算子对之间的交互，发现互补的算子逻辑

## 方法概述
G-LNS 利用 LLM 作为生成器，在进化框架中同时设计大邻域搜索的破坏（destroy）和修复（repair）算子。与以往仅进化单一启发式的方法不同，G-LNS 通过协作评估机制捕获算子对之间的交互关系，使发现的算子在结构破坏和重建上形成互补。

## 主要结果
- 在 TSP 和 CVRP 基准上显著超越 LLM AHD 方法
- 超越强经典求解器
- 发现的启发式在更低计算预算下达到近最优解
- 对不同和未见过的实例分布具有鲁棒泛化能力

## 复现指南
- 需要 LLM API 访问（用于算子生成）
- 在标准 TSP/CVRP 基准上评估
- 实现 LNS 框架 + 进化搜索循环

## 要点总结
1. LLM 不仅能写代码，还能设计组合优化算法
2. 协同进化耦合算子比单独进化更有效
3. LLM 生成的启发式可以超越手工设计的经典算法
